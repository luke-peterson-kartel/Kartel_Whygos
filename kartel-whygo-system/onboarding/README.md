## üöÄ Launch Sequence

Use these materials in this order for maximum impact:

### Before the Meeting

1. **Schedule** 30-minute kickoff meeting with all department heads
2. **Prepare** your demo environment:
   - Clear terminal history
   - Increase font size for readability
   - Have sample commands ready
3. **Build** slide deck using the outline
4. **Practice** the live demo (run through commands)
5. **Review** the script to internalize talking points

### During the Meeting

1. **Present** using the slide deck
2. **Demo** the system live (follow script)
3. **Answer** questions using Q&A section
4. **Set expectations** for Monday launch

### Immediately After Meeting

1. **Send** personalized emails (within 30 minutes)
   - Attach each person's cheat sheet
   - Use the team-email-guide template
2. **Record** video walkthrough
   - Follow the video script
   - Edit per post-production checklist
3. **Share** video link in Slack #announcements
4. **Pin** important resources in relevant Slack channels

### First Week Support

**Monday Morning**:
- Send reminder: "WhyGO update day! Check your cheat sheet or ping Luke."
- Be available all day for questions

**Wednesday Check-in**:
- Slack message: "Who's recorded updates? Who needs help?"
- Offer screen-share sessions for anyone stuck

**Friday Recap**:
- Share company dashboard in #leadership channel
- Celebrate early adopters
- Note any patterns in questions/blockers

**Next Monday**:
- Review in leadership meeting
- Share learnings from week 1
- Adjust process if needed

## üí° Tips for Success

**For the Meeting**:
- Keep it positive and energizing - this is exciting, not a chore!
- Show, don't just tell - live demo is crucial
- Address concerns head-on - "I know it's CLI for now, Slack bot coming soon"
- End with clear next steps

**For the Email**:
- Send immediately while meeting is fresh
- Personalize the greeting
- Attach the RIGHT cheat sheet for each person
- Make it scannable with headers and formatting

**For the Video**:
- Speak slowly and clearly
- Show commands typed out (don't just paste)
- Add visual callouts in editing
- Keep it under 10 minutes

**For Week 1**:
- Over-communicate - multiple reminders OK
- Be responsive - answer questions fast
- Celebrate early adopters - call out who's updating
- Collect feedback - what's working, what's not?

## üÜò Troubleshooting

**If someone is stuck**:
1. Screen-share session (5 min fix most issues)
2. Verify they're in correct directory
3. Check their person_id spelling
4. Walk through one update together
5. Send them the video link

**If adoption is low**:
1. Send additional reminders
2. Make it competitive ("Sales at 100%! Who's next?")
3. Show the value ("We caught 3 blockers early this week")
4. 1-on-1 follow-ups with non-updaters

**If technical issues arise**:
1. Document the error
2. Fix in the codebase
3. Announce fix to team
4. Update cheat sheets if commands change
